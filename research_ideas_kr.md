## 갭 분석 (Gap Analysis)

세 가지 연구 흐름을 종합 분석한 결과, 다음과 같은 핵심적인 연구 공백이 도출되었습니다:

1. **검색과 그라운딩은 여전히 분리된 태스크로 존재한다.** ColPali 계열 검색기는 올바른 페이지를 찾지만 페이지 내 어디에 정보가 있는지는 알려주지 못한다. 그라운딩 모델(Ferret, Florence-2, Grounding DINO)은 객체를 위치 추정할 수 있지만 대규모 코퍼스에 대한 검색 맥락에서는 동작하지 않는다.

2. **유일한 연결 고리는 사후적(post-hoc)이고 임시적(ad-hoc)이다.** 공간적 기반 문서 검색 논문(Georgiou, 2025)은 ColPali patch 점수를 OCR 영역에 추론 시점에 매핑하지만, 별도의 OCR 파이프라인이 필요하고, end-to-end로 학습되지 않으며, IoU@0.5에서 59.7%의 적중률에 불과하다.

3. **영역 수준(region-level) 검색은 태동기에 있으나 미성숙하다.** RegionRAG는 영역 단위 검색을 제안하지만 별도의 bounding box 지도 학습과 복잡한 이중 목적 손실이 필요하다. 단일 패스에서 검색과 위치 추정을 동시에 학습하는 모델은 없다.

4. **검색 + 위치 추정을 동시에 평가하는 벤치마크가 없다.** ViDoRe는 페이지 수준 nDCG를 측정한다. BBox-DocVQA는 단일 페이지에서의 그라운딩을 측정한다. "쿼리와 10,000개 페이지가 주어졌을 때, 올바른 페이지를 검색하고 올바른 영역을 가리킬 수 있는가?"를 평가하는 벤치마크는 존재하지 않는다.

5. **토큰 수준 patch 임베딩은 풍부하지만 공간 추론에 활용되지 못하고 있다.** ColPali는 페이지당 1024개의 patch 임베딩을 생성하며 이미지 영역과 공간적 대응 관계를 가지지만, 이 공간 구조는 late-interaction 스코어링에만 사용되고 위치 추정에는 활용되지 않는다.

6. **효율성이 우려 사항이다.** 다중 벡터 표현은 막대한 저장 공간을 필요로 한다(페이지당 1024 x 128차원). 프루닝 접근법(DocPruner, HPC-ColPali)은 저장 공간을 줄이지만 공간적으로 유의미한 patch를 파괴할 수 있다.

7. **문서 그라운딩 모델은 검색 인식이 부족하다.** Marten, BBox-DocVQA 베이스라인, VISA 같은 모델은 이미 검색된 문서에서 동작한다. 관련 문서를 찾는 것과 그 안에서 위치를 특정하는 것을 동시에 최적화하지 않는다.

## 연구 아이디어

### 아이디어 1: ColPali-Ground -- Patch 수준 공간 그라운딩을 갖춘 End-to-End 문서 검색

**한 줄 요약:** ColPali의 late-interaction 프레임워크를 확장하여 관련 문서 페이지를 검색하는 동시에 가장 관련성 높은 영역에 대한 bounding box를 출력하도록 하며, 검색-그라운딩 결합 목적 함수로 end-to-end 학습한다.

**동기:** ColPali는 공간적으로 구조화된 patch 임베딩을 생성하며 이는 정보가 페이지 어디에 있는지를 암묵적으로 인코딩한다. 각 쿼리 토큰과 모든 문서 patch 간의 late-interaction 점수(MaxSim)는 이미 소프트 공간 어텐션 맵을 계산한다. 그러나 이 공간 신호는 스코어링 후 버려지며 -- 집계된 페이지 수준 점수만 사용된다. 이는 낭비적이다: patch 수준 유사도는 추가 모델이나 OCR 파이프라인 없이도 관련 영역을 위치 추정하기에 충분한 정보를 포함한다. Georgiou(2025)의 사후적 접근법은 patch 점수를 OCR 영역에 전파하여 위치 추정을 달성할 수 있음을 보였지만, 이는 학습된 능력이 아닌 추론 시점의 임시 방편이었다. ColPali 계열 모델이 patch 점수가 검색과 위치 추정 양쪽에 사용될 것임을 명시적으로 인식하도록 학습한 기존 연구는 없다.

**방법:**
- 아키텍처: ColQwen2(Qwen2-VL 백본)에서 출발. 경량 위치 추정 헤드 추가 -- 상위-K 쿼리-patch 유사도 벡터(patch 그리드 상의 2D 위치 인덱스 포함)를 입력받아 정규화된 이미지 공간에서 bounding box 좌표 [x1, y1, x2, y2]를 회귀하는 소형 MLP.
- 학습 단계 1 (검색 사전학습): ViDoRe 학습 파이프라인의 쿼리-페이지 쌍에 대한 표준 ColPali 대조 학습. 강력한 patch 임베딩을 확립한다.
- 학습 단계 2 (검색 + 그라운딩 결합 미세조정): BBox-DocVQA, VISA (Wiki-VISA, Paper-VISA), 그리고 영역 어노테이션이 포함된 DocVQA/InfographicVQA의 새롭게 큐레이션된 하위 집합을 사용. 손실은: L = L_retrieval (대조 학습) + lambda * L_grounding (예측된 bounding box와 정답 bounding box 간의 smooth-L1, 검색된 양성 페이지에서만 계산).
- 핵심 혁신: 그라운딩 손실은 검색에 사용되는 동일한 patch 임베딩을 통해 역전파되어, 모델이 페이지 수준 랭킹과 하위 페이지 위치 추정 양쪽에 동시에 적합한 patch 표현을 학습하도록 강제한다. 이는 두 태스크가 동일한 late-interaction 임베딩 공간을 공유하는 형태의 다중 태스크 표현 학습이다.
- 추론: 주어진 쿼리에 대해, (1) 모든 인덱싱된 페이지에 대해 late-interaction 점수를 계산하여 상위-K 페이지를 검색하고, (2) 각 상위-K 페이지에 대해 쿼리-patch 유사도 행렬에서 위치 추정 헤드를 실행하여 bounding box를 예측한다. 유사도 행렬은 검색 시 이미 계산되므로 총 오버헤드는 최소한이다.

**기대 기여:** 단일 다중 벡터 임베딩 공간에서 검색과 그라운딩을 동시에 최적화하는 최초의 모델. ColPali 계열 patch 임베딩이 이중 역할을 수행할 수 있음을 입증한다. 새로운 평가 프로토콜을 확립한다: Retrieval@K + IoU@0.5 결합 지표.

**관련 연구 차별화:** 추론 시점에만 동작하고 OCR이 필요한 Georgiou(2025)와 다르다. 별도의 영역 제안과 다른 검색기를 사용하는 RegionRAG와 다르다. 결합 학습 없이 이미 검색된 문서에서만 귀속을 수행하는 VISA와 다르다. 검색 환경이 아닌 DocVQA를 위한 마스크 생성을 수행하는 Marten과 다르다.

**대상 학회:** CVPR 2027, ECCV 2026, NeurIPS 2026, ICLR 2027.

**실현 가능성:** 중간. BBox-DocVQA 데이터(이용 가능, 32K QA 쌍과 bounding box), VISA 데이터셋(이용 가능), ColPali 학습 인프라(오픈 소스)가 필요하다. 위치 추정 헤드는 단순한 추가이다. 주요 엔지니어링 노력은 더 많은 영역 어노테이션 데이터 큐레이션과 결합 학습 스케줄 설계이다. ColQwen2-2B 미세조정에는 단일 A100-80G 또는 H100이면 충분하다. 소요 기간: 3-4개월.

**위험 요소:** patch 그리드 해상도(예: 1024px 이미지에서 32x32)가 작은 텍스트 영역에 대한 위치 추정 정밀도를 제한할 수 있다. lambda를 신중하게 조정하지 않으면 그라운딩 손실이 검색 성능을 방해할 수 있다. 기존 데이터셋의 정답 bounding box가 patch 그리드 경계와 완벽하게 정렬되지 않아 보간이 필요할 수 있다.

### 아이디어 2: GroundedViDoRe -- 문서 검색과 영역 위치 추정의 결합 평가를 위한 벤치마크

**한 줄 요약:** 문서 검색 정확도와 공간적 위치 추정 품질을 동시에 평가하는 최초의 대규모 벤치마크를 구축하여, 그라운딩 검색 시스템의 공정한 비교를 가능하게 한다.

**동기:** 현재 이 분야에는 전체 검색 + 위치 추정 파이프라인을 포착하는 벤치마크가 없다. ViDoRe(V1/V2/V3)는 페이지 수준 nDCG만 평가한다. BBox-DocVQA는 단일 주어진 페이지에서의 그라운딩을 평가한다. VISA는 bounding box 어노테이션을 제공하지만 소규모 문서 집합에 대해서만이다. "10K+ 페이지와 자연어 쿼리가 주어졌을 때, 올바른 페이지를 찾고 올바른 영역을 가리킬 수 있는가?"라는 근본적인 질문을 던지는 벤치마크가 없다. 이러한 벤치마크 없이는 이 결합 태스크에 대한 진전을 측정하거나 비교할 수 없다.

**방법:**
- 데이터 구축 파이프라인 (BBox-DocVQA의 Segment-Judge-Generate에서 영감):
  1. 다양한 코퍼스 수집: 5개 도메인에서 50K 페이지 (arXiv 학술 논문, 재무 보고서, 기술 매뉴얼, 인포그래픽/슬라이드, 의학 문서). 기존 코퍼스 활용 (OpenDocVQA, ViDoRe V3 페이지, PubLayNet).
  2. 영역 추출: 문서 레이아웃 분석(DocLayout-YOLO 또는 Florence-2)을 실행하여 의미적 영역(텍스트 블록, 표, 그림, 헤더, 캡션)을 bounding box와 함께 검출.
  3. 쿼리 생성: 각 영역에 대해 강력한 VLM(GPT-4o 또는 Qwen2.5-VL-72B)을 사용하여 해당 특정 영역의 정보가 필요한 자연어 질문을 생성. REAL-MM-RAG 난이도 체계를 따라 3단계 난이도 생성: (a) 키워드 매칭(쉬움), (b) 패러프레이즈(중간), (c) 추론 필요(어려움).
  4. 사람 검증: 어노테이터가 쿼리-영역 쌍의 20%를 정확성 검증하고, bounding box를 조정하며, 모호한 쿼리를 필터링.
  5. 부정 페이지: 하드 네거티브 포함 (동일 문서 내 시각적으로 유사한 페이지, 관련되지만 답이 아닌 콘텐츠가 있는 페이지).
- 평가 지표:
  - 검색: nDCG@5, Recall@1, Recall@5 (페이지 수준)
  - 위치 추정: IoU@0.25, IoU@0.5, IoU@0.75 (정답 페이지의 bounding box)
  - 결합: Grounded-Recall@K = 정답 페이지가 상위-K에 포함되고 예측된 bounding box가 정답 영역과 IoU >= 임계값을 달성하는 쿼리 비율
  - 효율성: 쿼리당 지연 시간, 인덱스 크기
- 베이스라인 평가: (1) ColPali + 사후 OCR 전파 (Georgiou 방법), (2) ColPali 검색 + Florence-2 그라운딩 파이프라인, (3) Qwen2.5-VL을 이용한 Retrieve-then-Ground, (4) RegionRAG (이용 가능한 경우).

**기대 기여:** 근본적인 평가 공백을 채운다. 커뮤니티에 그라운딩 검색을 측정하는 표준화된 방법을 제공한다. 벤치마크 자체가 중요한 리소스 기여이다. 베이스라인 분석은 현재 시스템이 이 문제 해결에서 얼마나 멀리 있는지, 어떤 구성 요소가 병목인지를 드러낸다.

**관련 연구 차별화:** ViDoRe는 페이지 수준만 평가. BBox-DocVQA는 단일 페이지 그라운딩. VISA는 소규모이고 VQA 중심. MMDocBench는 세밀한 이해를 평가하지만 검색은 아님. GroundedViDoRe는 대규모 검색과 영역별 위치 추정 평가를 결합한 최초의 벤치마크이다.

**대상 학회:** NeurIPS 2026 Datasets and Benchmarks Track, CVPR 2027, ACL 2026, EMNLP 2026.

**실현 가능성:** 높음. 핵심 구성 요소가 이용 가능: 문서 코퍼스(ViDoRe V3, OpenDocVQA), 레이아웃 검출기(DocLayout-YOLO, Florence-2), 쿼리 생성(GPT-4o API), 평가 코드(MTEB 프레임워크, 표준 IoU 계산). 주요 비용은 사람 어노테이션(추정: 10K 쿼리-영역 쌍 검증에 200 어노테이터-시간, 약 $5K). 벤치마크는 점진적으로 구축 가능. 소요 기간: 4-5개월.

**위험 요소:** 모호한 레이아웃에 대한 어노테이터 간 일치도가 낮을 수 있다. VLM이 생성한 쿼리가 쉽게 설명 가능한 영역(예: 표와 그림)에 편향되어 텍스트 중심 영역을 과소대표할 수 있다. 모델이 개선됨에 따라 벤치마크가 빠르게 진부해질 수 있다. 완화책: 숨겨진 테스트 셋과 정기적으로 업데이트되는 리더보드를 포함한다.

### 아이디어 3: 공간 토큰 증류(Spatial Token Distillation) -- 텍스트 검색기에 VLM의 Patch 수준 공간 지식을 전달하기

**한 줄 요약:** ColPali 계열 patch 임베딩에 내재된 공간 위치 추정 지식을 경량 텍스트 전용 검색기로 증류하여, 추론 시 비전 인코더 없이 OCR+레이아웃 특성만으로 문서를 검색하고 bounding box를 예측할 수 있게 한다.

**동기:** ColPali 계열 다중 벡터 검색기는 모든 페이지를 비전 인코더(3B+ 파라미터)로 처리하고 페이지당 1024개 벡터를 저장해야 하며, 이는 기업 규모 배포(수백만 페이지)에 비용이 과다하다. 한편, DocLLM과 LayoutLLM 같은 레이아웃 인식 텍스트 모델은 OCR로부터 얻은 bounding box 좌표가 상당한 공간 정보를 포착할 수 있음을 보여준다. 핵심 통찰은: VLM 기반 검색기가 공간 이해의 최적 교사(teacher)이지만, 텍스트+레이아웃 모델이 비용의 일부로 공간 인식을 유지하는 훨씬 효율적인 학생(student)이 될 수 있다는 것이다. 이는 텍스트 기반 검색의 효율성과 비전 기반 검색의 공간 인식을 연결한다.

**방법:**
- 교사 모델: 각 문서 페이지에 대해 patch 임베딩 E_v in R^{N_patches x d}를 생성하는 학습된 ColQwen2 모델.
- 학생 모델: OCR 토큰 + bounding box 좌표(기존 OCR 파이프라인으로부터)를 입력으로 받아 토큰 임베딩 E_t in R^{N_tokens x d}를 생성하는 레이아웃 인식 인코더(ModernVBERT 아키텍처 또는 소형 LM).
- 공간 증류 손실: 각 OCR 토큰에 대해, 해당 토큰의 bounding box와 겹치는 비전 patch를 식별. 소프트 할당 행렬 A를 계산하며, A[i,j] = IoU(bbox_token_i, bbox_patch_j) / sum_k IoU(bbox_token_i, bbox_patch_k). 그러면 증류 손실은: L_distill = sum_i ||E_t[i] - sum_j A[i,j] * E_v[j]||^2. 이는 각 텍스트 토큰 임베딩이 자신의 공간 위치와 겹치는 비전 patch 임베딩의 가중 조합을 근사하도록 강제한다.
- 검색 손실: 학생의 다중 벡터 표현에 대한 표준 대조 학습 손실, late interaction(ColBERT 방식 MaxSim) 사용.
- 추론 시 위치 추정: 쿼리가 주어지면, 쿼리 토큰과 학생의 문서 토큰 임베딩 간 MaxSim을 계산. 최고 점수 문서 토큰은 이미 bounding box 좌표를 가지고 있다(OCR으로부터). 상위-K 스코어링 토큰을 공간적으로 클러스터링(bounding box 중심에 대한 DBSCAN)하여 영역 수준 bounding box를 생성.
- 학습 데이터: OCR+레이아웃 추출과 ColQwen2 patch 임베딩이 모두 있는 동일한 문서를 사용. 추가 어노테이션 불필요.

**기대 기여:** 공간 이해가 비전에서 텍스트+레이아웃 모달리티로 증류를 통해 전이될 수 있음을 입증한다. 텍스트 인코더 효율성으로 VLM에 근접한 위치 추정 품질을 달성한다(10-50배 빠르고, 10-100배 작은 인덱스). OCR 토큰과 비전 patch 간의 기하학적 대응을 존중하는 새로운 공간 증류 손실.

**관련 연구 차별화:** 더 작은 VLM을 처음부터 학습하는(여전히 비전 인코더 필요) ModernVBERT와 다르다. 학습 없이 추론 시 patch를 OCR에 매핑하는 Georgiou(2025)와 다르다. 레이아웃을 이해에는 사용하지만 검색에는 사용하지 않는 DocLLM과 다르다. VLM에서 텍스트 전용 모델로의 공간 검색 지식 증류는 최초이다.

**대상 학회:** ICLR 2027, ACL 2026, EMNLP 2026, NeurIPS 2026.

**실현 가능성:** 중상. 학습 코퍼스에서 ColQwen2로 patch 임베딩을 추출하는 것이 필요(일회성 비용, 병렬화 가능). OCR 파이프라인 필요(예: Tesseract, EasyOCR, PaddleOCR -- 모두 무료). 학생 모델은 소형(250M-1B 파라미터) 가능하며 단일 A100에서 학습 가능. 공간 할당 행렬 A는 사전 계산 가능. 소요 기간: 3-4개월.

**위험 요소:** OCR 오류가 공간 대응 관계를 저하시킬 수 있다. OCR 순서가 잘못되는 복잡한 레이아웃의 문서는 학생을 혼란시킨다. 이 접근법은 OCR 파이프라인에 의존하여 ColPali의 "OCR-free" 장점을 부분적으로 무효화한다. 완화책: 고품질 OCR 엔진을 사용하고 OCR 신뢰도가 낮은 학습 샘플을 필터링한다.

### 아이디어 4: 적응적 영역 줌을 활용한 계층적 다중 세분화 검색

**한 줄 요약:** 먼저 거친 표현으로 페이지 수준에서 검색한 다음, 관련 하위 영역으로 적응적 영역 수준 재검색을 수행하는 2단계 검색 시스템을 설계하며, 세분화 수준에 걸쳐 파라미터를 공유하는 단일 모델을 사용한다.

**동기:** 현재 검색 시스템은 고정된 세분화 수준에서 동작한다 -- 페이지 수준(ColPali, DSE) 또는 실험적으로 영역 수준(RegionRAG). 그러나 최적의 세분화 수준은 쿼리에 따라 다르다: "회사 설립 연도는?"은 표의 단일 셀이 필요할 수 있고, "방법론을 요약하라"는 전체 섹션에 걸친다. 적절한 줌 수준을 동적으로 선택하는 계층적 접근이 더 자연스럽고 효율적일 것이다. 이는 Ferret-v2의 임의 해상도 그라운딩과 R-VLM의 2단계 줌인 접근에서 영감을 받았다.

**방법:**
- 아키텍처: 공유 비전-언어 백본을 가진 단일 VLM 인코더(Qwen2-VL 또는 PaliGemma).
- 인덱싱: 각 문서 페이지에 대해 다중 세분화 인덱스 생성:
  - 수준 0 (페이지): 모든 patch 임베딩을 단일 벡터로 평균 풀링 (BiPali와 유사).
  - 수준 1 (사분면): 페이지를 2x2 그리드로 분할. 각 사분면의 patch를 평균 풀링하여 페이지당 4개 벡터.
  - 수준 2 (영역): 4x4 그리드로 분할하여 페이지당 16개 벡터.
  - 수준 3 (패치): 전체 ColPali 방식 1024개 벡터 (지연 저장 또는 상위 후보에만 적용).
- 검색 파이프라인:
  1. 단계 1 (거친 검색): 수준 0 벡터와 표준 ANN 인덱스(HNSW)로 상위-100 페이지 검색. 단일 벡터 bi-encoder이므로 빠름.
  2. 단계 2 (영역 위치 추정): 상위-100 페이지에 대해 수준 1 또는 수준 2 벡터를 로드. 사분면/영역 수준에서 late-interaction 점수 계산. 페이지를 재순위화하고 각 페이지 내 가장 관련성 높은 영역을 식별.
  3. 단계 3 (정밀 위치 추정, 선택적): 상위-10 페이지에 대해 수준 3(전체 패치) 벡터를 로드하거나 더 높은 해상도로 재인코딩. 상위 스코어링 패치를 클러스터링하여 정밀 bounding box를 생성하는 세밀한 MaxSim 실행.
- 학습: 다중 세분화 대조 학습. 각 수준에서 양성은 답을 포함하는 영역이고 음성은 같은 페이지의 다른 수준 영역(하드 페이지 내 음성)과 다른 페이지의 영역. 공유 백본이 수준 간 표현 일관성을 보장.
- 적응적 줌 정책: 어떤 줌 수준이 필요한지를 예측하는 경량 분류기(쿼리 임베딩에 대한 소형 MLP)를 학습하여, 쉬운 쿼리는 수준 1에서 멈추고 어려운 쿼리는 수준 3까지 진행. 이는 계산 비용을 분산시킨다.

**기대 기여:** 단일 모델로 다중 세분화에 걸쳐 동작하는 시각 문서를 위한 최초의 계층적 검색 시스템. 플랫 ColPali(항상 전체 패치 상호작용을 계산)나 플랫 bi-encoder(세밀한 정보를 잃음)보다 더 나은 지연시간-정확도 트레이드오프를 달성한다. 적응적 줌 정책은 검색 효율성에 대한 새로운 기여이다.

**관련 연구 차별화:** RegionRAG는 고정된 영역 세분화를 사용하며 별도의 영역 제안이 필요. ColPali는 플랫(페이지 수준만). HPC-ColPali와 DocPruner는 효율성을 위해 패치를 프루닝하지만 검색 세분화를 변경하지 않음. 이 연구는 원칙적인 다중 수준 계층을 도입. 줌인 아이디어는 Ferret-v2의 임의 해상도 그라운딩에서 착안하되 VQA가 아닌 검색에 적용.

**대상 학회:** CVPR 2027, NeurIPS 2026, ICLR 2027, SIGIR 2026.

**실현 가능성:** 중간. 다중 수준 인덱스 구축은 간단(다른 그리드 해상도에서의 평균 풀링). 주요 연구 과제는 다중 세분화 학습: 영역 수준 어노테이션이 필요하며, BBox-DocVQA, 레이아웃 검출 출력, 합성 어노테이션으로 얻을 수 있다. 적응적 줌 분류기가 복잡성을 추가한다. 추정 계산: 학습에 A100 2-4대. 소요 기간: 4-6개월.

**위험 요소:** 고정 그리드(2x2, 4x4)가 자연스러운 문서 영역과 정렬되지 않을 수 있다(예: 페이지 3/4를 차지하는 표). 완화책: 하드 그리드 대신 레이아웃 검출에 기반한 소프트 풀링 영역 사용. 다중 수준 인덱스는 총 저장 공간을 증가시키지만(수준 0-2에서 페이지당 1 + 4 + 16 = 21 벡터), 이는 ColPali의 1024 벡터보다 훨씬 적다. 다양한 쿼리 복잡도를 보여주는 데이터가 제한적이어서 적응적 정책 학습이 어려울 수 있다.

### 아이디어 5: 대조적 Patch-영역 정렬을 통한 검색 인식 그라운딩 사전학습

**한 줄 요약:** (a) 대규모 문서 코퍼스에서의 대조적 검색과 (b) 레이아웃 어노테이션을 활용한 patch-영역 정렬을 동시에 학습하는 새로운 목적 함수로 비전-언어 모델을 사전학습하여, 검색과 그라운딩 양쪽이 즉시 가능한 파운데이션 모델을 생성한다.

**동기:** 현재 VLM은 일반적인 비전-언어 이해(예: Qwen2-VL, PaliGemma)를 위해 사전학습되거나 검색(ColPali) 또는 그라운딩(Ferret, Florence-2)을 위해 별도로 미세조정된다. 검색 + 공간적 그라운딩의 결합 목적으로 처음부터 사전학습된 모델은 없다. 이는 검색과 그라운딩을 결합하려면 항상 별도 모델의 파이프라인이나 사후적 융합이 필요하다는 것을 의미한다. 두 목적 함수로 사전학습된 파운데이션 모델은 두 태스크 모두에 본질적으로 우수한 표현을 생성하여, 복잡한 다단계 파이프라인의 필요성을 제거할 것이다.

**방법:**
- 사전학습 데이터:
  - 검색 쌍: ViDoRe 학습 데이터, Wikipedia 스크린샷 + NQ 쌍(DSE 방식), 학술 논문 스크린샷 + 생성된 쿼리로부터 소싱된 10M+ 쿼리-페이지 쌍.
  - 레이아웃 어노테이션: 검색 코퍼스의 모든 페이지에 대해 레이아웃 검출기(Florence-2 또는 DocLayout-YOLO)를 실행하여 bounding box와 의미적 레이블(텍스트, 표, 그림, 헤더 등)을 가진 영역 추출.
- 사전학습 목적 함수 (3개 손실, 동시 최적화):
  1. **대조적 검색 손실** (L_ret): 쿼리 임베딩과 페이지 수준 문서 임베딩(평균 풀링된 patch 임베딩) 간 표준 배치 내 대조 학습 손실. 모델이 관련 vs. 비관련 페이지를 구별하도록 가르친다.
  2. **Patch-영역 정렬 손실** (L_align): 검출된 각 레이아웃 영역 R_k의 bounding box B_k에 대해, B_k 내부 patch의 평균 patch 임베딩을 계산. 이 영역 임베딩과 해당 영역의 텍스트 내용(OCR 또는 원본 텍스트로 추출) 간 코사인 유사도를 최대화. 개별 patch가 자신이 속한 레이아웃 영역의 내용을 인코딩하도록 가르친다.
  3. **공간 그라운딩 손실** (L_ground): 특정 영역을 참조하는 쿼리(예: "하단 절반의 표")가 주어지면, 모델이 올바른 patch를 선택해야 한다. 쿼리-관련 영역 bounding box와 patch 그리드 간의 겹침으로부터 도출된 정답을 기반으로 한 patch 선택에 대한 이진 교차 엔트로피 손실로 구현.
- 아키텍처: Qwen2-VL-2B에 임베딩을 위한 프로젝션 헤드와 경량 그라운딩 디코더(쿼리 토큰과 patch 임베딩 간 교차 어텐션, 이어서 bounding box 좌표를 예측하는 MLP).
- 총 손실: L = L_ret + alpha * L_align + beta * L_ground.

**기대 기여:** "검색-그라운딩 인식" patch 임베딩을 생성하는 최초의 사전학습 레시피. 문서 인텔리전스를 위한 새로운 유형의 파운데이션 모델을 창출하며, 위치 추정도 제공하면서 ColPali의 대체재 역할을 할 수 있다. 세 가지 목적 함수의 사전학습이 핵심 기술적 참신성이다.

**관련 연구 차별화:** ColPali는 검색만을 위해 학습. Florence-2는 그라운딩을 위해 사전학습하지만 검색은 아님. VDocRAG는 검색을 위한 사전학습을 제안하지만 공간적 그라운딩 없이. Marten은 보조적 마스크 생성을 추가하지만 검색이 아닌 VQA를 위해. 대조적 검색, 레이아웃 인식 패치 정렬, 공간적 그라운딩을 단일 사전학습 레시피에 결합한 것은 최초이다.

**대상 학회:** ICLR 2027, NeurIPS 2026, CVPR 2027.

**실현 가능성:** 중하. 가장 리소스 집약적인 아이디어. 10M+ 페이지에 대한 사전학습에 상당한 계산이 필요(추정: 에포크당 32-64 A100-시간, 3-5 에포크 필요). 전체 코퍼스에 대한 레이아웃 검출은 일회성 비용이지만 상당함. 그러나 모든 구성 요소가 개별적으로는 잘 이해되어 있다. 2B 모델과 1M 페이지로의 소규모 개념 증명은 학술 예산으로 실현 가능. 소요 기간: 5-7개월.

**위험 요소:** 세 가지 손실이 경쟁할 수 있다: 검색 손실은 판별적 페이지 수준 표현을 추구하고, 그라운딩 손실은 세밀한 공간 특이성을 추구한다. 이들이 자연스럽게 정렬되지 않을 수 있다. 신중한 손실 가중치와 커리큘럼(예: 검색으로 시작한 후 그라운딩 추가)이 핵심. 자동 생성된 레이아웃 어노테이션의 품질이 L_align에 영향을 미친다. 사전학습 데이터 규모가 이점을 완전히 실현하기에 불충분할 수 있다.

### 아이디어 6: 희소 그라운딩 토큰(Sparse Grounding Tokens) -- 문서 검색에서 학습 가능한 공간 앵커를 통한 효율적 위치 추정

**한 줄 요약:** ColPali의 밀집 1024-patch 표현을 공간 인식이 가능한 소수(예: 16-32개)의 학습 가능한 "그라운딩 토큰"으로 대체하여, 극적으로 축소된 인덱스 크기로 경쟁력 있는 검색을 달성하면서 각 토큰의 학습된 공간적 수용 영역을 통한 위치 추정을 가능하게 한다.

**동기:** 다중 벡터 문서 검색에서 핵심 긴장 관계는 공간적 풍부함(더 많은 벡터 = 더 나은 위치 추정)과 효율성(더 적은 벡터 = 더 빠른 검색, 더 작은 인덱스) 사이에 있다. DocPruner와 HPC-ColPali는 사후적으로 중복 패치를 프루닝하려 하지만, 이는 모델이 더 적은 토큰에 정보를 집중하도록 학습된 적이 없기 때문에 최적이 아니다. 한편, perceiver/resampler 아키텍처(Flamingo, BLIP-2에서 사용)는 학습 가능한 쿼리를 통해 시각 토큰을 압축하지만 공간적 대응 관계를 잃는다. 핵심 아이디어: (a) 각각 페이지의 공간 영역을 전문화하고 (b) 함께 검색과 위치 추정에 충분한 정보를 제공하는 학습 가능한 토큰을 학습하면 어떨까?

**방법:**
- 아키텍처: VLM 인코더(Qwen2-VL-2B)에서 출발. 비전 인코더가 위치 정보를 가진 N_patch 개의 patch 임베딩을 생성한 후, K개의 학습 가능한 공간 앵커 토큰(K=16 또는 32) 도입. 각 앵커 토큰 q_k는 이미지 위의 정규 그리드에 초기화된 학습 가능한 2D 위치 (mu_k, sigma_k)를 가진다.
- 공간 어텐션: 각 앵커는 모든 패치에 어텐딩하되, 위치 의존적 가우시안 어텐션 바이어스를 적용: attn_weight(q_k, patch_j) = softmax(q_k^T patch_j / sqrt(d) + gaussian(pos_j; mu_k, sigma_k)). 이는 각 앵커가 주로 자신의 공간 중심 근처 패치에 어텐딩하도록 장려하면서도 전역 정보 흐름을 허용한다.
- 1-2개의 교차 어텐션 레이어 후, K개의 앵커 임베딩이 페이지의 다중 벡터 표현이 된다(ColPali의 1024 패치 벡터를 대체).
- 학습: (1) 쿼리 토큰 임베딩과 K개 앵커 임베딩 간의 대조적 late-interaction 손실. (2) 공간 정규화 손실: 학습된 위치 (mu_k, sigma_k)가 페이지를 균일하게 커버하도록(균일 2D 분포와의 KL 발산 최소화) 앵커 붕괴를 방지. (3) 위치 추정 손실: 쿼리가 주어졌을 때, 학습된 위치 (mu_k)가 정답 영역에 가장 가까운 앵커가 가장 높은 유사도 점수를 가져야 한다. 이는 모델이 위치 추정에 앵커 위치를 사용하도록 가르친다.
- 추론 시 위치 추정: 쿼리 토큰과 앵커 임베딩 간 MaxSim 계산 후, 가장 높은 유사도 점수를 가진 앵커가 대략적인 위치(mu_k)를 드러낸다. sigma_k는 공간 범위를 제공. 추가 계산 없이 직접 위치 추정 히트맵을 생성한다.

**기대 기여:** 효율성-위치 추정 트레이드오프를 조화시키는 새로운 아키텍처. ColPali 대비 32-64배 압축(1024 대신 16-32 벡터)을 달성하면서 공간적 위치 추정 능력을 유지한다. 가우시안 어텐션을 가진 학습 가능한 공간 앵커는 공간 인식 표현 학습을 위한 새로운 아키텍처 프리미티브이다.

**관련 연구 차별화:** DocPruner와 HPC-ColPali는 공간 인식 없이 사후적으로 프루닝. Perceiver/resampler 아키텍처는 압축하지만 공간 구조를 잃음. ModernVBERT는 더 작은 모델을 사용하지만 여전히 많은 토큰을 생성. 이 연구는 공간적으로 의미 있는 토큰으로의 압축을 위해 특별히 설계된 아키텍처이다. DETR의 object query와 정신적으로 가장 가깝지만, 탐지가 아닌 문서 검색에 적용.

**대상 학회:** NeurIPS 2026, ICLR 2027, CVPR 2027, ECCV 2026.

**실현 가능성:** 중간. 아키텍처는 비교적 단순(위치 의존적 바이어스를 가진 교차 어텐션). 학습에는 ColPali와 동일한 데이터에 위치 추정 손실을 위한 영역 어노테이션이 추가로 필요. 주요 과제는 K=16-32 토큰이 검색 품질에 충분한지 확인하는 것(ColPali는 1024 사용). K에 대한 어블레이션 연구가 핵심. 2B 모델 학습에는 단일 A100이면 충분. 소요 기간: 3-5개월.

**위험 요소:** 16-32 토큰만으로는 작은 세부 사항에 대한 쿼리(예: 표 셀의 특정 숫자)에 필요한 세밀한 정보를 잃을 수 있다. 가우시안 어텐션 바이어스가 의미적으로 관련되지만 공간적으로 먼 패치에 어텐딩하는 모델의 능력을 과도하게 제약할 수 있다. 공간 정규화 손실이 검색 손실과 충돌할 수 있다(예: 대부분의 쿼리가 특정 페이지 영역에 대한 것이면, 모델은 앵커를 거기에 집중하려 하지만 정규화는 균일 커버를 강제). 이 접근법은 위치 추정이 단일 앵커 위치로 근사될 수 있다고 가정하지만, 여러 분리된 영역의 정보가 필요한 쿼리에는 성립하지 않을 수 있다.

### 아이디어 7: UnifiedRetGround -- 단일 순방향 패스에서 검색, 그라운딩, 답변을 수행하는 단일 자기회귀 VLM

**한 줄 요약:** 쿼리와 후보 문서 이미지 세트가 주어졌을 때, 단일 생성에서 (1) 가장 관련성 높은 페이지의 인덱스, (2) 근거 영역의 bounding box 좌표, (3) 답변을 출력하는 단일 자기회귀 비전-언어 모델을 학습하여, 검색, 그라운딩, 생성을 하나의 모델로 통합한다.

**동기:** 현재 그라운딩된 문서 QA에 대한 접근법은 3단계 파이프라인이다: 검색(ColPali) → 그라운딩(Ferret/Florence-2) → 생성(GPT-4o/Qwen2.5-VL). 각 단계는 자체 모델, 자체 오류, 자체 지연 시간을 가진다. 오류가 누적된다: 검색이 실패하면 그라운딩과 생성도 실패한다. Kosmos-2와 Qwen3-VL 같은 최근 모델은 단일 자기회귀 모델이 텍스트와 bounding box 좌표를 모두 생성할 수 있음을 보였다. 그리고 GENIUS 같은 모델은 생성적 접근이 검색(문서 식별자를 자기회귀적으로 생성)에 작동할 수 있음을 보인다. 야심적인 통찰은: 이 능력들이 통합될 수 있다는 것이다.

**방법:**
- 아키텍처: 강력한 VLM(Qwen2.5-VL-7B 또는 InternVL-7B)을 기본 모델로.
- 입력 형식: 모델은 텍스트 쿼리와 N개 후보 페이지 이미지(N=5-20, BiPali 같은 경량 1단계 검색기로 사전 필터링)를 받는다. 모든 페이지 이미지는 특수 <PAGE_1>, <PAGE_2>, ... 구분 토큰과 함께 다중 이미지 입력으로 연결된다.
- 출력 형식 (구조화된 생성): 모델은 JSON 유사 출력을 생성:
  ```
  {"page": 3, "bbox": [0.12, 0.45, 0.67, 0.82], "answer": "매출이 15% 증가했습니다"}
  ```
  페이지 인덱스는 답을 포함하는 입력 페이지를 선택한다. bbox 좌표는 정규화된 [0,1] 값이다. 답변은 텍스트 응답이다.
- 학습:
  - 단계 1: 기존 데이터셋(M3DocVQA, MP-DocVQA)에서 정답에 올바른 페이지 인덱스가 포함된 다중 페이지 QA 미세조정.
  - 단계 2: BBox-DocVQA, VISA 데이터셋에서 정답에 페이지와 bounding box가 모두 포함된 그라운딩 QA 미세조정.
  - 단계 3: 결합 손실로 동시 미세조정: 페이지 선택 토큰에 대한 교차 엔트로피 + bounding box 토큰에 대한 smooth-L1 + 답변 토큰에 대한 교차 엔트로피. 좌표를 위치 토큰으로 이산화하는(Kosmos-2 / Qwen-VL 방식) 표준 자기회귀 다음 토큰 예측 프레임워크에서 특수 토큰 유형 사용.
- 추론 파이프라인: BiPali가 상위-20 페이지를 검색(빠름, 단일 벡터). UnifiedRetGround가 이 20 페이지와 쿼리를 단일 순방향 패스에서 처리하여 페이지, bbox, 답변을 출력.

**기대 기여:** 단일 자기회귀 순방향 패스에서 페이지 선택, 공간적 그라운딩, 답변 생성을 수행하는 최초의 모델. 초기 필터링 단계 이후 별도의 검색 및 그라운딩 모델의 필요성을 제거한다. "검색-그라운딩-생성" 파이프라인이 단일 모델로 축소될 수 있음을 보여주어 배포를 단순화하고 오류 누적을 줄인다. 구조화된 생성 형식은 그라운딩된 문서 QA 시스템 구축을 위한 실용적 기여이다.

**관련 연구 차별화:** GENIUS는 그라운딩 없이 생성적 검색을 수행. Kosmos-2는 그라운딩을 하지만 검색은 아님. M3DocRAG는 ColPali + Qwen2-VL을 별도 모델로 사용. VISA는 귀속을 하지만 페이지 선택은 아님. ViDoRAG는 다중 에이전트 추론을 사용하지만 단일 통합 모델은 아님. 전체 파이프라인을 위한 최초의 end-to-end 모델이다.

**대상 학회:** ICLR 2027, NeurIPS 2026, ACL 2026, CVPR 2027.

**실현 가능성:** 중간. 핵심 질문은 7B VLM이 20개 페이지 이미지(각각 ~1024 토큰 = 총 20K 시각 토큰) + 쿼리를 단일 순방향 패스에서 안정적으로 처리할 수 있는가이다. 이는 다음이 필요할 수 있다: (a) 적극적 이미지 압축(낮은 해상도, 512px), (b) 강력한 긴 컨텍스트 모델(Qwen2.5-VL은 32K 토큰 지원), 또는 (c) N을 5-10 페이지로 축소. 학습에는 bounding box 어노테이션이 있는 다중 페이지 데이터셋이 필요하며, 존재하지만 증강이 필요할 수 있다. 계산: 7B 모델 미세조정에 A100 ~4대. 소요 기간: 4-6개월.

**위험 요소:** 가장 큰 위험은 컨텍스트 길이이다: 고해상도에서 20 페이지는 모델의 효과적 컨텍스트 윈도우를 초과하여 품질을 저하시킬 수 있다. 5 페이지로 줄이면 재순위화 능력이 제한된다. 모델이 지름길을 학습할 수 있다(항상 페이지 1 선택, 또는 일반적인 bounding box 생성). 구조화된 출력 형식은 퇴화된 출력을 방지하기 위해 신중한 학습이 필요하다. 또한, 진정한 대규모 검색(수백만 페이지)에서는 1단계 필터(BiPali)가 병목이 되며, 이 모델은 재순위화와 그라운딩 단계에만 도움이 된다.
